# profile.ts 編集ガイド

ポートフォリオの表示内容は、すべて `src/data/profile.ts` で管理しています。
このファイルを編集するだけで、画面の内容が変わります。

---

## 1. 基本項目（文字列を差し替えるだけ）

`'...'` の中身を自分の情報に書き換えます。

| 項目 | 内容 | 反映先 |
|------|------|--------|
| `name` | 名前（漢字） | Hero, Contact, Footer |
| `nameReading` | ふりがな | Contact |
| `navName` | ナビバー表示名 | Navbar |
| `location` | 住所（市区町村まで） | Contact |
| `email` | メールアドレス | Contact |
| `github` | GitHubユーザー名 | Contact, Projects |
| `greeting` | 挨拶文 | Hero |
| `subtitle` | 自己紹介文 | Hero |
| `tagline` | ステータスバッジ | Hero |
| `narrative` | 経歴の概要文 | About |

---

## 2. 経歴（careers）

経歴は `careers` の配列（`[...]` の中）にブロックを追加・削除します。
画面では全経歴が要約表示され、クリックで詳細が展開されます。

### 経歴を追加する（詳しく書く場合）

既存の `},` の後ろに以下をコピーして貼り付け、内容を書き換えます。

```ts
        {
            role: '事務',
            desc: '総務部で社内システムの運用を担当。',
            project: '社内システム移行',
            tasks: [
                'マニュアル作成',
                'ユーザー問い合わせ対応',
            ],
            achievements: [
                '問い合わせ件数を30%削減',
            ],
            tech: 'Excel, Google Workspace',
        },
```

各項目の意味:

| 項目 | 必須 | 説明 |
|------|:----:|------|
| `role` | 必須 | 職種（例: 営業、事務、エンジニア） |
| `desc` | 必須 | 業務内容の概要（1-2文） |
| `company` | 任意 | 会社名（書かなければ表示されない） |
| `project` | 任意 | プロジェクト名 |
| `tasks` | 任意 | 主な担当業務（箇条書き） |
| `achievements` | 任意 | 実績・成果（箇条書き） |
| `tech` | 任意 | 使用ITスキル（カンマ区切り） |

### 経歴を追加する（簡潔に書く場合）

`role`, `desc` の2項目だけでOKです。

```ts
        {
            role: 'アルバイト',
            desc: '接客業務を担当。',
        },
```

### 経歴を削除する

`{` から `},` までのブロックを丸ごと削除します。

### 表示順を変えたい場合

配列の上にあるものが画面でも上に表示されます。
ブロックの順番を入れ替えれば表示順が変わります。

## 3. スキル（skillCategories）

スキルはカテゴリごとにまとめて表示されます。

### カテゴリを追加する

```ts
        {
            title: 'バックエンド',
            skills: [
                { name: 'Python', level: 1 },
                { name: 'SQL', level: 2 },
            ] as Skill[],
        },
```

### 既存カテゴリにスキルを追加する

該当カテゴリの `skills` 配列に1行追加します。

```ts
                { name: 'Vue.js', level: 1 },
```

### level の意味

| level | 意味 | 目安 |
|:-----:|------|------|
| `1` | 基礎 | 学習中・基本的な使い方は分かる |
| `2` | 実務 | 業務や制作で実際に使った |
| `3` | 得意 | 自信を持って使える |

### カテゴリ・スキルを削除する

該当のブロックまたは行を丸ごと削除します。

---

## 4. 資格（certifications）

### 資格を追加する

```ts
        { name: '日商簿記2級', date: '2023.03' },
```

強調表示したい場合は `highlight: true` を付けます。

```ts
        { name: '応用情報技術者', date: '2025.04', highlight: true },
```

### 資格を削除する

該当の行を丸ごと削除します。

## 5. 制作物（projects）

### 制作物を追加する

```ts
        {
            title: '家計簿アプリ',
            description: '月ごとの収支を可視化するアプリ。',
            image: 'https://images.unsplash.com/photo-xxxxxxx',
            tags: ['React', 'Firebase'],
            github: 'https://github.com/yourname/kakeibo',
            demo: '',
            time: '30h',
        },
```

各項目の意味:

| 項目 | 必須 | 説明 |
|------|:----:|------|
| `title` | 必須 | 制作物名 |
| `description` | 必須 | 説明文（1-2文） |
| `image` | 必須 | 画像URL |
| `tags` | 必須 | 使用技術タグ |
| `github` | 必須 | GitHubリポジトリURL（空 `''` でボタン非表示） |
| `demo` | 必須 | デモURL（空 `''` でボタン非表示） |
| `time` | 必須 | 制作時間（例: "30h"） |
| `featured` | 任意 | `true` で大きいカード表示 |
| `challenge` | 任意 | 課題（書くとケーススタディ表示になる） |
| `approach` | 任意 | アプローチ |
| `result` | 任意 | 成果 |

### 制作物を削除する

`{` から `},` までのブロックを丸ごと削除します。

---

## 6. セクション自体を非表示にしたい場合

配列を空にすると、そのセクションの内容が非表示になります。

```ts
    careers: [] as Career[],
    certifications: [] as Certification[],
```

---

## 7. 編集時によくあるエラーと対処

画面が真っ白になった / エラーが出た:

- カンマ `,` の付け忘れ・重複がないか確認
- 文字列が `'...'` で囲まれているか確認（`"..."` でもOK）
- `[` と `]`、`{` と `}` の対応が合っているか確認
- 不安な場合は、変更前の状態に戻してから1項目ずつ追加してみてください

開発サーバー（`npm run dev`）が動いている状態で保存すると、即座に画面に反映されます。
エラーが起きた場合もターミナルにメッセージが出るので、参考にしてください。
