# profile.ts 編集ガイド

ポートフォリオの表示内容は、すべて `src/data/profile.ts` で管理しています。
このファイルを編集するだけで、画面の内容が変わります。

このガイドは、`README.md` の起動手順を完了し、`npm run dev` で画面が開いている状態で使う想定です。
最初は `name` を1か所だけ変えて、保存後に画面の名前表示が変わることを確認してください。

## 1. 基本項目（文字列を差し替えるだけ）

`'...'` の中身を自分の情報に書き換えます。
どこを編集するか: `src/data/profile.ts` の基本情報エリアです。
保存後にどこを見るか: 画面上部の自己紹介、基本情報、フッターを確認します。

| 項目 | 内容 | 反映先 |
|------|------|--------|
| `name` | 名前（漢字） | Hero, Contact, Footer |
| `nameReading` | ふりがな | Hero |
| `navName` | ナビバー表示名 | Navbar |
| `location` | 住所（市区町村まで） | Contact |
| `email` | メールアドレス | Contact |
| `github` | GitHubユーザー名 | Contact, Footer |
| `greeting` | 挨拶文 | Hero |
| `subtitle` | 自己紹介文 | Hero |
| `tagline` | ステータスバッジ | Hero |
| `narrative` | 経歴の概要文 | About |

## 2. 経歴（careers）

経歴は `careers` の配列（`[...]` の中）にブロックを追加・削除します。
各カードには職種（`role`）と概要（`desc`）が常に表示され、クリックで詳細（担当業務・実績・スキル等）が展開されます。
どこを編集するか: `careers: [ ... ]`
保存後にどこを見るか: 「職務経歴」セクションのカード一覧と詳細表示です。

### 経歴を追加する

最初は、既存の1件を丸ごとコピーして中身を書き換える方法が安全です。
既存の `},` の後ろに以下をコピーして貼り付け、内容を書き換えます。

```ts
        {
            role: '事務',
            desc: '総務部で社内システムの運用を担当。',
            project: '社内システム移行',
            tasks: [
                'マニュアル作成',
                'ユーザー問い合わせ対応',
            ],
            achievements: [
                '問い合わせ件数を30%削減',
            ],
            tech: 'Excel, Google Workspace',
        },
```

各項目の意味:

| 項目 | 必須 | 説明 |
|------|:----:|------|
| `role` | 必須 | 職種（例: 営業、事務、エンジニア） |
| `desc` | 必須 | 業務内容の概要（1-2文）。トグルを開かなくても常に見える |
| `company` | 任意 | 会社名（書かなければ表示されない） |
| `project` | 必須 | プロジェクト名。トグルを開くと表示される |
| `tasks` | 必須 | 主な担当業務（箇条書き）。トグルを開くと表示される |
| `achievements` | 必須 | 実績・成果（箇条書き）。トグルを開くと表示される |
| `tech` | 必須 | 使用ITスキル（カンマ区切り）。トグルを開くと表示される |

### 経歴を削除する

`{` から `},` までのブロックを丸ごと削除します。

### 表示順を変えたい場合

配列の上にあるものが画面でも上に表示されます。
ブロックの順番を入れ替えれば表示順が変わります。

## 3. スキル（skillCategories）

スキルはカテゴリごとにまとめて表示されます。
どこを編集するか: `skillCategories: [ ... ]`
保存後にどこを見るか: 「スキル・資格」セクションです。

### カテゴリを追加する

```ts
        {
            title: 'バックエンド',
            skills: [
                { name: 'Python', level: 1 },
                { name: 'SQL', level: 2 },
            ] as Skill[],
        },
```

### 既存カテゴリにスキルを追加する

該当カテゴリの `skills` 配列に1行追加します。

```ts
                { name: 'Vue.js', level: 1 },
```

### level の意味

| level | 意味 | 目安 |
|:-----:|------|------|
| `1` | 基礎 | 学習中・基本的な使い方は分かる |
| `2` | 実務 | 業務や制作で実際に使った |
| `3` | 得意 | 自信を持って使える |

### カテゴリ・スキルを削除する

該当のブロックまたは行を丸ごと削除します。

## 4. 資格（certifications）

どこを編集するか: `certifications: [ ... ]`
保存後にどこを見るか: 「スキル・資格」セクション内の「保有資格」です。

### 資格を追加する

```ts
        { name: '日商簿記2級', date: '2023.03' },
```

強調表示したい場合は `highlight: true` を付けます。

```ts
        { name: '応用情報技術者', date: '2025.04', highlight: true },
```

### 資格を削除する

該当の行を丸ごと削除します。

## 5. 制作物（projects）

どこを編集するか: `projects: [ ... ]`
保存後にどこを見るか: 「制作物」セクションのカードです。

### 制作物を追加する

最初は、既存の1件を丸ごとコピーして中身を書き換える方法が安全です。

```ts
        {
            title: '家計簿アプリ',
            description: '月ごとの収支を可視化するアプリ。',
            image: 'https://images.unsplash.com/photo-xxxxxxx',
            tags: ['React', 'Firebase'],
            github: 'https://github.com/yourname/kakeibo',
            demo: '',
            time: '30h',
        },
```

各項目の意味:

| 項目 | 必須 | 説明 |
|------|:----:|------|
| `title` | 必須 | 制作物名 |
| `description` | 必須 | 説明文（1-2文） |
| `image` | 必須 | 画像URL。外部URLでも `/images/xxx.webp` のようなローカル画像でもOK（ローカル画像は `public/images/` に置く） |
| `tags` | 必須 | 使用技術タグ |
| `github` | 任意 | GitHubリポジトリURL。項目は残し、中身を空 `''` にするとボタン非表示 |
| `demo` | 任意 | デモURL。項目は残し、中身を空 `''` にするとボタン非表示 |
| `time` | 必須 | 制作時間（例: "30h"） |
| `featured` | 任意 | `true` で大きいカード表示 |
| `challenge` | 任意 | 課題（書くとケーススタディ表示になる） |
| `approach` | 任意 | アプローチ |
| `result` | 任意 | 成果 |

### 制作物を削除する

`{` から `},` までのブロックを丸ごと削除します。

## 6. セクション自体を非表示にしたい場合

配列を空にすると、そのセクションの中身は消えます。
ただし、見出しやセクション枠が残る場合があります。

```ts
    careers: [] as Career[],
    certifications: [] as Certification[],
    projects: [] as Project[],
```

注意:

- `careers: []` にしても「職務経歴」の見出しは残ります
- `certifications: []` にしても「スキル・資格」セクション自体は残ります
- `projects: []` にしても「制作物」の見出しは残ります

セクションごと完全に消したい場合は、`src/App.tsx` で該当コンポーネントを外します。

## 7. オリジナリティを出すカスタマイズ

テンプレートをそのまま使うと他の人と同じ見た目になります。
ここでは「自分らしさ」を出すためのカスタマイズ方法を紹介します。

### 7-1. テーマカラーを変える

編集ファイル: `src/index.css`

サイト全体の色は CSS 変数で管理されています。
`:root` 内の2行を変えるだけで、アクセントカラーとグラデーションが一括で変わります。

```css
  --accent-primary: #2563eb;   /* メインカラー（デフォルト: 青） */
  --accent-secondary: #7c3aed; /* サブカラー（デフォルト: 紫） */
```

カラー例:

| テーマ | accent-primary | accent-secondary |
|--------|---------------|-----------------|
| 青×紫（デフォルト） | `#2563eb` | `#7c3aed` |
| 緑×青 | `#059669` | `#0891b2` |
| オレンジ×ピンク | `#ea580c` | `#db2777` |
| モノクロ | `#374151` | `#6b7280` |

ダークモードの色も変えたい場合は、同ファイル内の `[data-theme="dark"]` ブロックも同様に編集します。

### 7-2. フォントを変える

編集ファイル: `src/index.css`

ファイル1行目の Google Fonts URL と、CSS 変数を変更します。

```css
/* 1行目: Google Fonts のURLを差し替え */
@import url('https://fonts.googleapis.com/css2?family=Noto+Sans+JP:wght@400;500;700&display=swap');

/* :root 内のフォント変数を変更 */
  --font-sans: 'Noto Sans JP', sans-serif;    /* 本文フォント */
  --font-heading: 'Noto Sans JP', sans-serif;  /* 見出しフォント */
```

[Google Fonts](https://fonts.google.com/) で好みのフォントを探して差し替えてください。

### 7-3. セクションの順番を変える

編集ファイル: `src/App.tsx`

`<main>` 内のコンポーネントの並び順を入れ替えるだけです。

```tsx
      <main>
        <Hero />
        <Projects />   {/* 制作物を先に見せたい場合 */}
        <About />
        <Skills />
        <Contact />
      </main>
```

注意: ナビバーのリンク順も合わせる場合は `src/components/Navbar.tsx` の `navLinks` 配列の順番も変更してください。

### 7-4. セクションを削除・追加する

**削除**: `src/App.tsx` から該当の行を消します。

```tsx
        {/* <Skills /> */}   {/* コメントアウトで非表示にもできる */}
```

**追加**: 新しいコンポーネントを作り、`App.tsx` に追加します。
既存のコンポーネント（例: `src/components/Skills.tsx`）をコピーして改変するのが手軽です。

### 7-5. ダークモードの初期値を変える

編集ファイル: `src/App.tsx`

デフォルトはライトモードです。ダークモードを初期値にする場合:

```ts
  const [theme, setTheme] = useState<'light' | 'dark'>(() => {
    return (localStorage.getItem('theme') as 'light' | 'dark') ?? 'dark'; // 'light' → 'dark'
  });
```

## 8. 編集時によくあるエラーと対処

画面が真っ白になった / エラーが出た:

- カンマ `,` の付け忘れ・重複がないか確認
- 文字列が `'...'` で囲まれているか確認（`"..."` でもOK）
- `[` と `]`、`{` と `}` の対応が合っているか確認
- 不安な場合は、変更前の状態に戻してから1項目ずつ追加してみてください

開発サーバー（`npm run dev`）が動いている状態で保存すると、即座に画面に反映されます。
エラーが起きた場合もターミナルにメッセージが出るので、参考にしてください。
